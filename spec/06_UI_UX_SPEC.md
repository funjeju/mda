# 06. UI/UX 스펙

MDA의 화면·플로우·인터랙션을 정의한다.

---

## 6.1 UX 북극성

사용자 경험의 3대 원칙:

1. **2탭 이내 입력** — 모든 입력은 위젯 또는 홈 화면에서 2탭 이내
2. **같은 데이터, 다른 시각** — 뷰 전환은 1탭
3. **AI가 먼저 제안, 사용자는 승인** — "확인"이 "입력"보다 많게

---

## 6.2 정보 구조 (Information Architecture)

### 앱 (모바일)

```
┌─ 하단 탭 ─────────────────────────┐
│ 🏠 홈  📊 대시  💬 대화  ⚙️ 설정   │
└────────────────────────────────────┘

🏠 홈
  ├── 오늘의 입력창 (상시 노출)
  ├── 오늘의 뷰 (사용자 선택: 투두/만다라트/캘린더)
  ├── 최근 AI 확인 요청
  └── 저녁 보고서 미리보기

📊 대시 (Dashboard)
  ├── 프로젝트 리스트
  │   └── 각 프로젝트 → 상세 (만다라트/간트/칸반)
  ├── 일기 보관함
  ├── 관계 (PersonContact)
  └── 지난 보고서

💬 대화
  ├── 새 대화 시작
  ├── 프로젝트 설계 모드
  ├── 저녁 회고 모드
  └── 최근 대화 기록

⚙️ 설정
  ├── 프로필 & 페르소나
  ├── 뷰 & 디자인 (다꾸)
  ├── 알림
  ├── 외부 연동
  ├── 프라이버시
  └── 구독
```

### 웹

```
┌─ 사이드바 ──────────────────┐
│ 🏠 홈                       │
│ 📁 프로젝트                  │
│   ├── Q1 마케팅              │
│   ├── 제주 미션북            │
│   └── + 새 프로젝트          │
│ 📅 캘린더                    │
│ 📊 대시보드                  │
│ 💬 대화                      │
│ 📓 일기                      │
│ 👥 관계                      │
│ 🧩 연동                      │
│ ⚙️ 설정                      │
└─────────────────────────────┘

메인 영역: 선택된 페이지 렌더
우측 패널 (optional): AI 컴패니언 (항상 접근 가능한 채팅)
```

## 6.3 페르소나별 첫 경험

### Light User 온보딩

```
1. 가입 후 첫 화면
   "안녕하세요! 하루를 어떻게 기록하고 싶으세요?"
   [💭 가볍게 메모] [📝 일정 관리] [🎯 목표 달성] [👥 팀과 함께]
         ↓
   "가볍게 메모" 선택 시 → Light 페르소나 자동 설정

2. 첫 입력 유도
   "오늘 뭐 하셨어요? 음성으로도 적어도 돼요."
   [음성 버튼] [텍스트 입력]

3. AI가 첫 입력 처리 시연
   사용자: "오늘 카페에서 친구 만남, 저녁에 운동 갈 예정"
   AI: "친구분 만나셨네요! 저녁 운동은 몇 시쯤이에요?"
       [19시] [20시] [나중에 알려드릴게요]

4. 다꾸 소개
   "오늘의 기록을 꾸며볼까요?"
   [스티커 선택] [스킵]

5. 위젯 설치 유도
   "홈 화면에 MDA 버튼을 두면 더 편해요"
   [설치 방법 보기] [나중에]
```

### Medium User 온보딩

```
1. "어떤 일을 관리하고 싶으세요?"
   [💼 일과 일상] — Medium 선택

2. "지금 하고 있는 프로젝트가 있나요?"
   사용자: "클라이언트 A 브랜딩 진행 중"
   → AI 러프 분해 → 미니 프로젝트 차트 즉시 생성 시연

3. "노션/캘린더 연동하시면 더 편해요"
   [노션 연결] [캘린더 연결] [나중에]

4. 기본 뷰 선택
   "어떤 뷰가 편하세요?"
   [투두 리스트] [만다라트] [캘린더]
   (미리보기 카드 표시)
```

### Heavy User 온보딩

```
1. "팀 규모는 어떻게 되세요?"
   [혼자] [2~5명] [6명 이상]

2. "팀 계정 만들기 / 혼자 시작 / 기존 팀 참여"

3. 팀 생성 시:
   - 팀 이름
   - 팀 멤버 초대
   - 기본 워크플로우 선택

4. "진행 중인 프로젝트를 가져올까요?"
   [노션 임포트] [Jira 임포트] [새로 시작]

5. 기본 뷰: 간트 또는 팀 만다라트
```

### 피벗 전환 애니메이션

섹션 → 팀 피벗 전환 시:

1. 현재 셀들이 중심으로 모임 (1초)
2. 새 축으로 재배열 (1초)
3. 셀 라벨 업데이트 (0.5초)

총 2.5초 내 완료. 같은 시각 언어 유지로 인지 부하 최소.

### 뷰 전환

상단 토글 또는 스와이프:

```
[📋 리스트] [📐 만다라트] [📅 캘린더] [📊 간트]
```

데이터는 동일, 렌더러만 교체. 선택된 태스크·필터는 유지.

## 6.6 인터랙션 패턴

### 제스처

| 제스처 | 동작 |
|--------|------|
| 탭 | 선택 / 열기 |
| 롱 프레스 | 컨텍스트 메뉴 |
| 스와이프 오른쪽 | 완료 표시 |
| 스와이프 왼쪽 | 수정 / 삭제 |
| 핀치 | 만다라트 줌 |
| 드래그 | 셀 이동 / 담당자 변경 |
| 더블 탭 | 다꾸 편집 모드 |

### 마이크로 인터랙션

- **태스크 완료**: 체크박스 → 폭죽 이모지 → 페이드 (총 1초)
- **섹션 완성**: 섹션 전체에 특수 효과 (별가루 낙하)
- **프로젝트 완성**: 전체 화면 축하 오버레이 (스킵 가능)
- **새 입력**: 입력창에 물결 효과
- **AI 처리 중**: 로딩 인디케이터 (뇌 모양 펄스)
- **오류**: 가벼운 흔들림 + 햅틱 피드백

### 드래그 앤 드롭

- 만다라트 셀 간 태스크 이동
- 캘린더 시간 블록 조정
- 간트 의존관계 드래그로 연결
- 다꾸 스티커 셀에 붙이기

## 6.15 페르소나 전환 UX

사용자가 페르소나를 바꿀 수 있어야 함 (성장하거나 필요 변화):

```
[Light] 가볍게 기록       ◯

  │ 미리보기             │
  │ (각 페르소나 샘플)   │
  └────────────────────┘

[적용]
```

전환 시 앱 재시작 없이 즉시 반영.
