# 15. 수익 모델 (Monetization)

MDA의 수익 구조. 하이브리드 — 개인 Freemium + B2B Team.

---

## 15.1 수익 원칙

1. **가치를 주고 받는다** — 공짜로 부담 주지 않는다
2. **진입 장벽 낮게** — Free 티어로 핵심 가치 체험
3. **업그레이드 유도는 필요 시** — 강제 모달·팝업 지양
4. **팀엔 팀 가격** — 개인과 팀은 다른 가치 제공
5. **다꾸 경제** — 감성 가치의 추가 수익화

---

## 15.2 티어 구조 (4단계)

### Free (무료)

**타겟**: Light 페르소나, 입문자

**포함**:
- 모든 Phase 1 핵심 기능
- 월 100k AI 토큰
- 녹음 파일 월 30분 (Whisper STT)
- 최대 3개 프로젝트
- 독립 태스크 무제한
- 기본 다꾸 (기본 스티커팩 + 3개 테마)
- 개인 팀 (1인)
- 외부 연동: Google Calendar만
- 저녁 보고서
- 광고 없음

**제한**:
- 팀 생성 불가 (혼자만)
- 대시보드 제한적
- 히스토리: 30일
- 대용량 파일 제한 (10MB)

### Plus ($7.99/월, ₩9,900)

**타겟**: Medium 페르소나, 개인 프로듀서

**Free + 추가**:
- 월 1M AI 토큰
- 녹음 무제한
- 프로젝트 무제한
- 모든 테마 + 계절 스티커팩
- 외부 연동 전체 (Calendar, Notion, Gmail, Photos, Health)
- 대용량 파일 (100MB)
- 프리미엄 스티커팩 30% 할인
- 주간·월간 리포트
- 히스토리: 무제한
- 우선 지원

### Team ($12.99/사용자/월, ₩15,900)

**타겟**: Heavy 페르소나, 5~50명 팀

**Plus + 추가**:
- 팀 생성 무제한
- 팀원 초대 무제한 (seat 당 과금)
- 팀 대시보드
- 팀 피벗 (assignee axis)
- 댓글 & 멘션
- 자동 스탠드업
- 주간 회고
- Slack 봇
- GitHub / Jira 연동
- 관리자 콘솔
- 감사 로그 (30일)

**최소**: 3 seats ($38.97/월~)

### Business (Enterprise, 맞춤 가격)

**타겟**: 50명 이상 조직

**Team + 추가**:
- SAML SSO
- E2E 암호화
- 감사 로그 1년
- 데이터 잔류 보장 (한국 리전)
- SLA 99.9%
- 전담 CSM
- 커스텀 통합
- 온프레미스 옵션 (논의)
- 교육·온보딩 지원

**가격**: $25/사용자/월~ (협의)

## 15.7 B2B 판매 전략

### 타겟 세그먼트

1. **1인 프리랜서 에이전시** (5~20명)
2. **한국 스타트업** (10~50명)
3. **교육 기관** (MDA Edu 특화)
4. **로컬 크리에이터** (1인 콘텐츠 제작자)

### 판매 채널

- **셀프서브**: 웹사이트에서 결제
- **Inbound Sales**: 문의 → 데모 → 계약
- **Outbound**: 타겟 리스트 직접 접촉 (Phase 3)
- **파트너**: 컨설팅사·디지털 트랜스포메이션 에이전시

### B2B 판매 프로세스

```
Lead 접수 (데모 요청)
  ↓
디스커버리 콜 (30분)
  ↓
데모 & 요구 매칭
  ↓
POC (2주 무료)
  ↓
제안서 & 계약
  ↓
온보딩 (전담 CSM)
  ↓
확장 (upsell)
```

### 엔터프라이즈 기능

- SAML SSO
- SCIM (자동 사용자 프로비저닝)
- 도메인 캡처 (자동 가입)
- Dedicated Infrastructure (선택)
- 데이터 잔류 (한국 리전)

## 15.15 결제 시스템 구현

### 주요 Flow

```
User clicks "Upgrade to Plus"
  ↓
Next.js: Stripe Checkout Session 생성
  ↓
Redirect to Stripe Checkout
  ↓
User completes payment
  ↓
Stripe Webhook → Cloud Function
  ↓
Firestore: user.subscription_tier = 'plus'
  ↓
Custom claim 업데이트
  ↓
Firebase Rules에서 Plus 기능 접근 허용
```

### Webhook 이벤트 처리

- `checkout.session.completed` → 구독 시작
- `customer.subscription.updated` → 플랜 변경
- `customer.subscription.deleted` → 취소
- `invoice.payment_failed` → 결제 실패

### 구독 상태 필드

```typescript
interface UserSubscription {
  tier: 'free' | 'plus' | 'team' | 'business';
  status: 'active' | 'past_due' | 'canceled' | 'paused';
  stripe_customer_id: string;
  stripe_subscription_id: string;
  current_period_end: Timestamp;
  cancel_at: Timestamp | null;
  seat_count: number;
}
```

## 15.16 팀 결제 특수 처리

### Seat 관리

- 관리자 콘솔에서 seat 조정
- 추가 초대 시 자동 seat 증가
- 제거 시 다음 청구 주기에 반영
- Pro-rated 청구

### 청구 담당자

- 팀 오너가 기본 청구 담당
- 별도 관리자 지정 가능 (Billing Admin)
- 회사 청구 이메일 별도 설정

### 결제 방식

- 신용카드 (Stripe)
- 세금계산서 (Business, 연 결제)
- 은행 송금 (Business, 협의)

## 15.19 윤리적 수익화

### 다크 패턴 금지

- 취소 버튼 숨기기 금지
- 자동 갱신 알림 필수
- 환불 요청 쉽게
- 데이터 내보내기 항상 가능

### 투명성

- 가격 변경 시 30일 전 고지
- 기능 이동 시 (Free → Plus) 명확히
- 기존 사용자 권리 존중 (grandfathering)

### 공정성

- 장애 시 프로라이트 환불
- 개인정보는 수익화하지 않음
- 광고 게재 금지 (영구)
