# 10. 협업 시스템 (Collaboration)

팀 기반 협업 기능. 개인도 "1인 팀" 구조로 동일하게 동작.

---

## 10.1 설계 원칙

1. **Team이 1급 시민** — 모든 데이터는 team_id 기준
2. **개인도 팀** — 가입 시 "개인 팀" 자동 생성
3. **유연한 권한** — 역할별 + 리소스별 세분화
4. **팀 간 투명성** — 일기는 사적, 업무는 공유
5. **초대 친화적** — 외부인도 쉽게 참여

---

## 10.2 Team 구조

### 팀 타입

| 타입 | 설명 | 특징 |
|------|------|------|
| `personal` | 1인 팀 (자동 생성) | 초대 불가, 본인만 |
| `collaborative` | 일반 팀 | 다인 초대, 협업 |

### 팀 생성

사용자 가입 시:
1. User 생성
2. Personal Team 자동 생성
3. TeamMember로 owner 등록

추가 팀 생성:
- 사용자가 "새 팀 만들기"
- 팀 이름·설명·초기 멤버 설정
- Stripe 구독 (팀 플랜 이상)

### 팀 전환

```
┌─────────────────────────┐
│ [▼ 내 워크스페이스]      │  ← 현재 팀
│                          │
│ 📌 개인                  │
│ ✓ 내 워크스페이스       │  ← 선택
│   사이드 프로젝트 팀    │
│   3명 가족 팀            │
│                          │
│ [+ 새 팀 만들기]        │
└─────────────────────────┘
```

전환 시 `user.current_team_id` 업데이트. 모든 데이터 쿼리는 이 필드 기준.

## 10.3 역할 (Roles)

### 역할 정의

| 역할 | 권한 |
|------|------|
| `owner` | 팀 삭제·결제·모든 권한 |
| `admin` | 멤버 초대·프로젝트 관리 |
| `member` | 태스크 생성·담당 |
| `viewer` | 읽기만 (무료 티어 확장용) |

### 권한 매트릭스

| 액션 | Owner | Admin | Member | Viewer |
|------|-------|-------|--------|--------|
| 팀 삭제 | ✓ | ✗ | ✗ | ✗ |
| 결제 관리 | ✓ | ✗ | ✗ | ✗ |
| 멤버 초대 | ✓ | ✓ | ✗ | ✗ |
| 멤버 제거 | ✓ | ✓ | ✗ | ✗ |
| 프로젝트 생성 | ✓ | ✓ | ✓ | ✗ |
| 프로젝트 삭제 | ✓ | ✓ | 소유자만 | ✗ |
| 태스크 생성 | ✓ | ✓ | ✓ | ✗ |
| 태스크 완료 | ✓ | ✓ | 담당자 | ✗ |
| 읽기 | ✓ | ✓ | ✓ | ✓ |
| 일기 쓰기 | ✓ | ✓ | ✓ | ✗ (자기 일기만) |

### 프로젝트별 오버라이드

프로젝트 단위로 개별 권한 부여 가능:

```typescript
interface ProjectMembership {
  project_id: string;
  user_id: string;
  role: 'owner' | 'collaborator' | 'viewer';
  // 팀 역할과 별개
}
```

예: Member인 사용자가 특정 프로젝트의 Owner가 될 수 있음.

## 10.4 초대 플로우

### 초대 방식

1. **이메일 초대** — 가장 기본
2. **링크 초대** — 공유 가능 링크 (만료 설정)
3. **QR 코드** — 오프라인 미팅에서

### 초대 UI

```
┌─────────────────────────────┐
│ 👥 멤버 초대                │
├─────────────────────────────┤
│ 이메일 주소:                │
│ [______________]            │
│                              │
│ 역할:                       │
│ [Member ▼]                  │
│                              │
│ 메시지 (옵션):              │
│ [_________________]         │
│                              │
│ [초대 보내기] [링크로]      │
└─────────────────────────────┘
```

### 수락 플로우

```
이메일 받음
  ↓
[수락 버튼] → MDA 앱/웹
  ↓
기존 계정? → 로그인 → 팀 합류
없으면? → 가입 → 팀 합류
  ↓
환영 화면 + 팀 온보딩
```

### 링크 초대 옵션

- 유효기간: 7일/30일/무제한
- 사용 횟수: 1회/무제한
- 기본 역할: Member (변경 가능)

## 10.5 담당자 배정 (Assignment)

### 배정 UX

태스크 생성 시:
```
담당자: [나 ▼]
        ├── 나
        ├── 민수
        ├── 수진
        └── 없음 (미배정)
```

### 자동 배정 힌트 (AI)

음성에서 "민수가 보고서 써야 해"라고 하면:
- AI가 민수를 후보로 제시
- 사용자 1탭 승인

### 배정 변경

- 태스크 상세에서 드롭다운
- 만다라트에서 드래그 앤 드롭으로 다른 담당자에게 이동
- 권한: 배정자·팀 Admin·프로젝트 Owner만 가능

### 담당자 변경 알림

- 새 담당자: "민수님이 '보고서 작성' 담당으로 지정됨"
- 이전 담당자: "수진님에게 이관됨"

## 10.6 팀 피벗 (Assignee Pivot)

`02_CORE_CONCEPTS.md`의 다축 피벗 중 하나.

### 팀 만다라트

```
┌────────────┬────────────┬────────────┐
│ 👤 민수    │ 👤 수진    │ 👤 펀제주  │
│ 5 tasks    │ 3 tasks    │ 7 tasks    │
│ ▓▓▓▓░ 80% │ ▓▓░░░ 40% │ ▓▓▓░░ 60% │
├────────────┼────────────┼────────────┤
│ 👤 영진    │ 🎯 Q1      │ 👤 지훈    │
│ 2 tasks    │ 마케팅     │ 4 tasks    │
│            │            │            │
├────────────┼────────────┼────────────┤
│ 👤 재호    │ 👤 미연    │ 👥 미배정  │
│ 3 tasks    │ 6 tasks    │ 2 tasks    │
```

각 셀 탭 → 해당 팀원의 만다라트 (섹션 피벗).

### 활용

- 팀원: 내 일과 동료 일 비교
- 미배정 부채 시각화

## 10.8 멘션 & 댓글

### 멘션

```
태스크 댓글:
"이 부분 @민수 확인 부탁"
```

@ 입력 시 팀원 자동완성. 멘션 받은 사람에게 알림.

### 댓글 UX

태스크 상세 하단:

```
💬 댓글 (3)

수진 · 오늘 14:15
컨셉은 좋은데 컬러가 너무 밝지 않아요?

펀제주 · 오늘 14:20
수진님 의견 반영해 3가지 버전 만들어볼게요

민수 · 오늘 14:22  
@펀제주 금요일 전까지 부탁드려요!

[댓글 입력...]  [파일 첨부]
```

### 댓글의 Entity

```typescript
interface Comment extends BaseEntity {
  task_id: string | null;
  project_id: string | null;
  journal_entry_id: string | null;
  
  author_id: string;
  content: string;
  mentions: string[];  // userIds
  attachments: string[];
  
  replies: Comment[];  // 중첩 (1단계)
  
  reactions: { [emoji: string]: string[] };  // emoji → userIds
}
```

## 10.10 스탠드업 자동화 (Phase 2)

### 아침 스탠드업

매일 09:00 (팀 설정):

각 팀원에게 질문:
```
☕ 좋은 아침! 오늘의 스탠드업

어제 한 일: [자동 입력 — 어제 완료 태스크]
오늘 할 일: [오늘 예정 태스크 목록]

[제출]  [수정]
```

제출된 내용 → 팀 피드에 요약 카드.

### 주간 회고 자동 준비

금요일 18:00 트리거:
- 이번 주 완료 태스크
- 진행률
- 이슈/장애물 (댓글에서 추출)
- 다음 주 예정

회의 전 자동 생성 → 회의에서 바로 사용.

### 사용 케이스

## 10.12 일기 · 관계의 팀 내 공유

### 기본 원칙

- 일기: 기본 private, 명시적으로 공유 시만 팀원 접근
- 관계: team_id 기준 저장되지만, 개인 PersonContact는 본인만

### 팀 일기 (옵션)

일부 팀은 "팀 일기" 기능 사용:
- 팀 회고 공유
- 감사 일기 (동료 칭찬)
- 이벤트 기록

이건 별도 공유 채널로 운영 (`shared_journal_id` 필드).

## 10.13 외부 협업자 (Phase 2)

### Guest 역할

정식 팀원은 아니지만 특정 프로젝트에만 접근:

```
프로젝트 상세 > 협업자 > 외부 추가
- 이메일 입력
- 역할: Guest (Viewer + 해당 프로젝트만)
- 만료일 설정
```

- 클라이언트에게 진행 상황 공유
- 외주 디자이너에게 특정 태스크만 오픈
- 컨설턴트 자문

## 10.15 보안 & 프라이버시 (팀)

### Firestore Security Rules

```javascript
// 팀 데이터 접근
match /teams/{teamId}/{document=**} {
  allow read: if isTeamMember(teamId);
  allow write: if isTeamMember(teamId) && hasWritePermission();
}

function isTeamMember(teamId) {
  return request.auth.uid in get(/teams/$(teamId)/members).data.userIds;
}
```

### 감사 로그 (Business Plan)

- 모든 쓰기 액션 로깅
- 누가·언제·무엇을·어디서
- 30일 (기본) / 1년 (Business) 보관
- 팀 소유자만 접근

## 10.16 1인 사용자의 "팀" 경험

개인 사용자도 Team 구조를 쓰지만:

- 팀 선택 UI 숨김 (혼자라서 의미 없음)
- 멤버 관리 UI 숨김
- "나의 공간"이라고 표시 (Team이라는 단어 미노출)

Medium/Heavy 페르소나가 팀을 만들면 그때 팀 UI 노출.
