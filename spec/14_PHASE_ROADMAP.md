# 14. Phase 로드맵

MDA 개발은 3단계로 진행한다. 각 Phase의 목표·기능·완료 기준을 정의한다.

---

## 14.1 Phase 철학

### Phase 1 — "혼자 쓸만함"

**목표**: 제작자(펀제주) 본인이 매일 사용할 수 있는 수준
- 북극성 문장의 핵심 가치 제공
- 입력 → 자산 → 보고서의 루프 완성
- 이쁘지 않아도 작동

**기간**: 2~3개월

### Phase 2 — "주변에 자랑할만함"

**목표**: 가족·지인·팀에 베타 배포 가능
- 협업 완성
- 디자인 완성도
- 다양한 연동
- 베타 유저 피드백 반영

**기간**: 3~4개월

### Phase 3 — "서비스로 판매할만함"

**목표**: 공개 서비스 출시
- 수익 모델 구동
- 엔터프라이즈 대응
- 확장성 검증
- 마케팅 지원

**기간**: 4~6개월

## 14.7 Phase별 인력 · 예산

### Phase 1

- 개발: 제작자(펀제주) + Claude Code + 1 보조 (프론트엔드 선택적)
- 디자인: 파트타임 디자이너 (Figma 시스템)
- 예산: 개인 자원 + Firebase/AI $200/월

### Phase 2

- 개발: 2~3명
- 디자인: 풀타임 1명
- 예산: 소규모 투자 또는 정부 지원금

### Phase 3

- 개발: 5~8명
- 디자인: 2명
- 마케팅: 1명
- CS: 1~2명
- 예산: Seed 투자 $500K~$1M

## 14.8 리스크 & 대응

### Phase 1 리스크

- AI 품질 부족 → 프롬프트 반복 개선, Claude fallback
- 위젯 구현 난이도 → Phase 2로 일부 이연 가능
- 제작자 burnout → 범위 축소 옵션 준비

### Phase 2 리스크

- 베타 피드백 방향성 충돌 → 북극성 기준 의사결정
- 인력 확보 → 주니어 채용 + Claude Code 활용
- 비용 급증 → 캐싱·최적화

### Phase 3 리스크

- 경쟁자 등장 → 차별화 재확인, 다꾸 생태계 활용
- 시장 반응 미지 → 니치 먼저 집중
- 기술 부채 누적 → Phase별 리팩토링 주간

## 14.9 성공 시나리오

### 낙관적 (Best Case)

- Phase 1에서 제작자 본인 만족 → 자발적 전파
- Phase 2에서 베타 유저 100명 확보, NPS 50+
- Phase 3에서 MAU 10K, ARR $500K
- 니치 도메인 (MDA Edu 등) 확장 성공

### 현실적 (Realistic)

- Phase 1 3개월 → 내부 사용
- Phase 2 6개월 → 지인 베타 50명
- Phase 3 1년 → 유료 구독 200명, ARR $50K
- 지속 운영 가능 수준

### 비관적 (Pessimistic)

- Phase 1 지연 → 범위 축소, MVP 간소화
- Phase 2 베타 반응 미지근 → 피봇 검토
- Phase 3 시장 진입 어려움 → 도메인 특화 (MDA Edu 등)로 전환

## 14.10 "접기 전략"

만약 Phase 2 이후 시장 반응이 저조하면:

**Plan B**: 특정 도메인 집중
- 가장 반응 좋았던 페르소나·업종에 집중 (예: 1인 프리랜서 전용)
- 범용성 축소, 전문성 강화

**Plan C**: 도메인 특화 매출
- MDA Edu 단일 제품으로 피봇 (교사용)
- MDA Creator (크리에이터용)

**Plan D**: Claude Code 학습 자산
- 개발 과정 자체를 콘텐츠화
- 오픈소스화
