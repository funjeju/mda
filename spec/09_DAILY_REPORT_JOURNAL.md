## 9.1 저녁 보고서 (Daily Report)

### 정의

매일 정해진 시간에 AI가 그날의 모든 입력을 종합해 생성하는 문서.

**목적**:
- 사용자에게 하루의 성취감 제공
- 놓친 것 없는지 재점검
- 내일 대비
- 장기적으로 회고 자산 축적

### 생성 시점

- **기본**: 21:00 (사용자 설정)
- **수동**: 사용자가 "지금 정리" 요청
- **조건부**: 오늘 입력이 3개 이상일 때만 자동 생성
- **조건 미달 시**: "오늘은 조용한 하루였어요" 미니 요약

### 생성 파이프라인

```
[매일 21:00 Cron]
    ↓
[오늘 입력 수집]
  - DailyEntries (today)
  - Completed tasks (today)
  - Journal entries (today)
  - Contact mentions
  - External events (calendar, photos)
    ↓
[Gemini 2.0 Flash 호출]
  - 상세 프롬프트 (04_AI_AGENT_SPEC 참조)
    ↓
[DailyReport entity 생성]
    ↓
[푸시 알림 전송]
    ↓
[사용자 확인]
  - 전체 보기
  - 수정
  - 공유 (옵션)
```

### 보고서 구조

```typescript
interface DailyReport {
  report_date: Timestamp;
  
  one_liner: string;          // "오늘의 한 줄"
  
  business: {
    completed: Task[];
    in_progress: Task[];
    added: Task[];
    tomorrow_preview: Task[];
    project_progress: ProjectDelta[];
    highlights: string[];     // AI 선정 키 이벤트
  };
  
  journal: {
    entries: JournalEntry[];
    emotion_timeline: { time: string; emotion: Emotion }[];
    met_contacts: PersonContact[];
    thoughts: string[];
    reflection_question: string | null;
  };
  
  reminders: { date: Timestamp; description: string }[];
  
  source_traces: DailyEntry[];
}
```

### UI (상세)

앞서 `06_UI_UX_SPEC.md` 6.8에 정의. 요점:

- 한 줄 요약이 제일 크게
- 비즈니스/일기 섹션 접을 수 있음
- 각 항목에서 원본 드릴다운
- 편집 가능 (잘못 분류된 것 이동)

### 편집 플로우

사용자가 보고서에서 항목 길게 누르면:

```
[이동] - 다른 섹션/프로젝트로
[수정] - 텍스트 편집
[삭제] - 잘못 인식된 것 제거
[원본 보기] - 어디서 왔는지
```

### 공유 기능 (Phase 2)

- 스크린샷 + 마스킹 (민감 정보 가림)
- SNS 공유용 포맷 (Instagram story 비율)
- 텍스트 복사
- PDF 내보내기 (유료)

## 9.4 일기 시스템

### 철학

**업무 관리가 아니라 마음 관리**. 업무 태스크와 같은 DB에 있지만 톤·UI·프라이버시가 다르다.

### 일기 엔트리 생성 경로

1. **AI 자동 감지** — 음성/텍스트에서 감정·일상 이벤트 감지 시
2. **직접 작성** — 일기 탭에서 수동 작성
3. **저녁 대화** — AI가 하루 끝에 질문해서 생성

### AI 감지 기준

다음 키워드/패턴 시 `journal_*` intent로 분류:

- 감정 표현: "좋다", "피곤", "기분", "화나", "행복" 등
- 일상 이벤트: "만났다", "갔다", "먹었다", "봤다"
- 반성/회고: "생각해보니", "오늘은", "이런 날"
- 기록 의도: "적어둬야지", "기록"

### 프라이버시

**기본값: 본인만 볼 수 있음** (`is_private: true`)

팀 플랜에서도 일기는 개인 영역. 팀원에게 노출되지 않음.

단, 사용자가 명시적으로 공유 선택 시:
- 특정 팀원에게 공유
- 공개 (팀 전체)

### 감정 분석

```typescript
interface EmotionAnalysis {
  primary: Emotion;
  intensity: number;
  secondary: Emotion[];
  triggers: string[];       // AI 추정 원인
  context: string;          // 상황
  is_concerning: boolean;   // 위기 신호
}
```

**감정 키워드 사전** (한국어):

```
joy:      기쁘, 좋, 행복, 즐거, 신나, 뿌듯, 만족
calm:     평온, 차분, 고요, 안정, 편안
excited:  설레, 신나, 기대, 흥분
anxious:  불안, 걱정, 초조, 긴장, 떨려
sad:      슬프, 우울, 눈물, 외로, 쓸쓸
angry:    화나, 짜증, 분노, 열받
tired:    피곤, 지쳐, 힘들, 무기력
grateful: 감사, 고마운, 덕분
proud:    자랑, 뿌듯, 해냈
```

### 위기 신호 감지

```typescript
const CONCERNING_PATTERNS = [
  /죽고\s*싶/,
  /살기\s*싫/,
  /사라지고\s*싶/,
  /희망\s*없/,
  /의미\s*없/,
  // ... (심리 전문가 자문 필요)
];

function checkConcerning(text: string): boolean {
  return CONCERNING_PATTERNS.some(p => p.test(text));
}
```

감지 시:
- `is_concerning: true` 플래그
- 즉시 공감 응답 + 지원 리소스
- 업무 자산 생성 일시 보류
- 사용자에게 확인 ("지금 괜찮으신가요?")

### 지원 리소스 메시지 예

```
"오늘 힘드셨나 봐요. 혼자 감당하기 어려울 때는 
전문가의 도움을 받는 것도 좋아요.

· 자살예방상담전화: ☎ 1393 (24시간)
· 청소년전화: ☎ 1388
· 정신건강위기상담: ☎ 1577-0199

저는 할 수 있는 만큼 들어드릴게요."
```

## 9.5 관계 관리 (Contact Intelligence)

### 철학

인간관계는 "데이터베이스"가 아니라 "기억". MDA는 기억을 돕되 대체하지 않는다.

### PersonContact 자동 생성

사용자 입력에서 인물 언급 감지 시:

```
"오늘 민수랑 카페에서 만났어."
  ↓
[Entity Extractor]
  people: ["민수"]
  ↓
[Contact Resolver]
  기존 "민수" 있음 → 업데이트
  없음 → 새로 생성 제안
  ↓
[사용자 확인] (신규일 때만)
  "'민수'라는 연락처를 새로 만들까요?"
  [예] [아니오, 같은 사람 선택] [스킵]
```

### 별칭 처리

같은 사람을 다르게 부르는 경우:

```
PersonContact {
  name: "김민수"
  alternate_names: ["민수", "민수형", "Mike"]
}
```

### 자동 맥락 누적

매 언급 시 `recent_contexts`에 추가:

```
{
  date: "2026-04-21",
  context: "카페에서 만남",
  source_entry_id: "entry-xxx"
}
```

최근 10개 유지. 이상은 아카이브.

### 관계 강도 (Implicit)

- 언급 빈도
- 마지막 만남 시점
- 함께한 프로젝트 수
- 일기에 등장 빈도

AI가 이를 종합해 "오래 연락 안 하셨네요" 같은 리마인드 생성.

### 리마인드 생성

3가지 경로:

1. **사용자 명시** — "2주 뒤 연락해야지"
2. **AI 감지** — "다음주에 만나자고 했는데 확정 안 됨"
3. **소홀함 감지** — 과거 빈번한 연락이었는데 최근 뜸함

### 리마인드 UX

```
┌─────────────────────────────┐
│ 💌 김민수님                  │
│ 2주 전에 "다음에 보자"고     │
│ 했어요. 연락해보실래요?     │
│                              │
│ [메시지 보내기] [1주 미루기] │
│ [연락함] [중요하지 않음]    │
└─────────────────────────────┘
```

### 프라이버시

- 연락처는 본인만 접근
- 전화번호·이메일은 암호화 저장
- 삭제 시 관련 일기에서 언급 유지 (일기 원본은 성역)
- 연락처 삭제 vs 관계 종료: 둘 다 지원

### 가족·팀 특수 처리

- 가족 태그: "가족" 태그로 묶어서 특별 알림 ("어머니 생신 D-3")
- 팀원: MDA 사용자면 자동 연결 (linked_user_id)

## 9.6 통합: 하루의 자산

### 모든 것이 연결된다

```
DailyEntry (입력)
    ↓
├─ Task (업무)
├─ JournalEntry (일기)
│   ├─ 언급: PersonContact
│   └─ 첨부: Photo
└─ PersonContact (관계)
    └─ Reminder → Task
```

### Daily Report에서 통합 뷰

저녁 보고서는 이 전체를 한 화면에 녹여 보여줌:

```
오늘의 한 줄
"민수와 의미 있는 대화"

업무:
- Q1 리포팅 완료

일기:
- 오후 카페에서 깊은 얘기

관계:
- 민수 — 2주 만에 만남

리마인드:
- 다음주 수요일 미팅
```

### 검색 통합

"제주 미션북" 검색 시:
- 관련 태스크
- 관련 일기 ("제주 답사 갔을 때...")
- 관련 사람 ("가족들과 함께")
- 관련 파일

## 9.8 AI Companion 대화 (저녁 회고)

### 자동 트리거

21:00 알림 탭 후:
- 보고서 열기 전 "오늘 어땠어?" 질문
- 사용자 답변 시 대화 시작

### 대화 패턴

```
AI: "오늘 많이 바쁘셨네요. 제일 기억에 남는 순간이 있어요?"

User: "민수랑 얘기한 게 좋았어"

AI: "민수님이 요즘 뭐 하시던가요?"

User: "새 프로젝트 시작한다고..."

AI: "도와주고 싶으신 게 있으세요?"

User: "응, 조언 좀 하고 싶어"

AI: "좋네요. '민수 프로젝트 조언하기'를 일정에 넣어둘까요?"

...
```

AI는 **절대 분석·판단·조언하지 않고** 경청·호기심·기록만.

### 저녁 회고 vs 주간 회고

- 저녁: 하루의 사실 + 감정 확인
- 주간 (선택): 패턴·인사이트 공유
- 월간 (선택): 더 깊은 성장 대화

## 9.9 감정 트렌드 대시보드

### 월별 감정 곡선

```

│ 이번 달 감정 흐름           │
│                              │
│ 😊 ▮▮▮▮▮▮▮▮▮▮▮▮▮▮▮▮▮ 45%  │
│ 😌 ▮▮▮▮▮▮▮▮▮▮▮▮ 30%      │

│                              │

│ 가장 힘들었던 날: 4/10      │
```

### 관찰

- 주중 vs 주말 패턴

**이건 단순 인사이트 제공, 판단이나 조언은 하지 않음.**
